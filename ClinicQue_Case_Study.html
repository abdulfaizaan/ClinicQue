ClinicQue

Curing Waiting Room Anxiety Through Real-Time Transparency

1. The Problem

Currently, over 76% of India's 1.5 million neighborhood clinics operate using chaotic paper token slips and human shouting. Patients sit in crowded spaces for 2-3 hours with zero visibility into their actual wait times, leading to severe frustration and "waiting room anxiety."

Meanwhile, clinic receptionists are forced to manage high-volume, multi-doctor queues entirely from memory, leading to skipped tokens, double-bookings, and immense operational stress.

"The moment you click 'Call Next Token' and the patient's phone instantly bursts with a notification—that's when clinic owners realize they aren't just managing a queue, they are curing waiting room anxiety."

2. The Solution

ClinicQue is a hyper-fast, zero-refresh digital queue manager architected specifically for multi-doctor clinics. It bridges the gap between software and the physical waiting room by combining real-time WebSocket synchronization with automated hardware integrations.

3. Key Architectural Innovations

⚡ Zero-Refresh Synchronization

Built on an event-driven Socket.IO architecture. State mutations on the Receptionist Dashboard are broadcasted instantly to TV Kiosks and Patient Smartphones within milliseconds. No HTTP polling is required.

🧠 Smart ETA Math Engine

Replaced hardcoded guesses with a dynamic algorithm. The backend calculates a rolling average of the last 5 real consultation times, seamlessly adjusting the math when a doctor goes on a lunch break.

🗣️ Automated Voice Calling

Integrated the native Browser Speech Synthesis API. When a token is called, the TV Kiosk automatically announces the patient's name and room number over the clinic speakers, eliminating human shouting entirely.

🛡️ Enterprise Security & Persistence

Backed by a multi-tenant SQLite database in WAL mode to ensure queues survive server crashes. Protected by IP-rate-limiting to prevent malicious actors from flooding the queue via QR codes.

4. Edge Case Handling

  • The "Lunch Break" Problem: If a doctor takes a 30-minute break, the ETA math would normally break. ClinicQue introduces a "Doctor on Break" pause state that natively halts the timestamp calculations, preserving the integrity of the historical average.
  • Receptionist Panic Clicking: The server strictly enforces a 2-second debounce cooldown on the "Call Next" socket event, preventing accidental double-skipping.
  • Patient No-Shows: Receptionists can instantly skip patients. The system drops them from the queue without injecting a 1-second consultation into the rolling average math.

5. Tech Stack

React 19 Node.js Express Socket.IO (WebSockets) SQLite3 Tailwind CSS v4 Framer Motion Browser Speech API

6. Business Impact & Outcomes

By moving from a single global array to an isolated, Multi-Doctor database schema, ClinicQue evolved from a basic hackathon toy into a viable B2B SaaS product.

The system achieves sub-10 second triage times for receptionists, and introduces zero-cost WhatsApp notifications via native wa.me protocol integrations—delivering massive value to clinic owners without incurring Twilio API costs.
